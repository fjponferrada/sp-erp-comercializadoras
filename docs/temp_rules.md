
130. **Regla 130 (Procesamiento de XMLs de Facturación F1)**: Los archivos F1 (Facturación) se ingieren en la tabla `F1Invoice` y se cruzan mediante raw SQL contra la tabla `Invoice` por código fiscal o número de factura. Un error 500 en la importación de XML F1 desde el worker local no implica que el XML sea "basura" o esté vacío, sino que comúnmente se debe a latencias de disco o red durante la lectura asíncrona; el parser nativo de Next.js es sólido frente a nodos de `<FacturaATR>` y `<OtrasFacturas>`, y la robustez contra estos fallos temporales se delega en reintentos programados (retries) desde el script de Node.js del scraper local.

131. **Regla 131 (Cálculo y Perdón de Decomisiones 2.0TD)**: En liquidaciones de comisiones con modalidad Upfront, todo contrato `2.0TD` exige una permanencia estricta de 365 días. Si el cliente cesa el servicio prematuramente (ej: día 331), el sistema resta financieramente los 34 días incumplidos prorrateados. La única salvedad inquebrantable a esta regla la compone el límite de un Euro (`decommissionAmount >= 1`). El motor de liquidación perdonará penalizaciones por bajas en los ultimísimos días cuya cuantía total a devolver sea inferior a 1,00 €, evitando ensuciar contablemente la liquidación con reclamos de escasos céntimos.
